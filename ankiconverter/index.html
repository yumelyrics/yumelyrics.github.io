Cara export .apkg dari Anki
  1. Buka Anki di komputer → menu File → Export
  2. Pilih Export format: Anki Deck Package (.apkg)
  3. Pilih deck yang mau di-convert
  4. Opsi "Support older Anki versions" boleh dicentang atau tidak — tool ini sudah support file .apkg dari versi Anki manapun (lama maupun baru)
  5. Klik Export, lalu upload hasilnya ke sini ✨

ℹ️ Catatan: tool ini cuma convert teks kartu (termasuk nama file gambar/audio kalau ada, mis. [sound:xxx.mp3]). File gambar/audio-nya sendiri gak ikut ke-extract ke CSV.

🔑 Opsi "Sertakan GUID": nambahin kolom identitas asli tiap kartu. Kalau CSV hasil convert ini (yang GUID-nya masih utuh, jangan diubah/dihapus) di-convert BALIK ke .apkg pakai mode "CSV → Apkg" di tool ini, kartunya bakal pakai ID yang sama kayak aslinya — jadi kompatibel buat update-in-place, bukan bikin duplikat.

Mau update kartu lama di Anki (bukan bikin duplikat)?

Ada 2 cara:

Cara 1 — Import manual di Anki (paling aman):

  1. Edit dulu isi kartunya di tabel preview di atas (mis. field terjemahan) sebelum download
  2. Download CSV-nya, lalu buka Anki di komputer → File → Import (bukan lewat tool ini)
  3. Pilih file CSV-nya, pastikan Note type & Deck yang dipilih SAMA kayak deck lama
  4. Aktifin opsi "Update notes that match first field" (nama opsinya bisa beda dikit tergantung versi Anki)
  5. Anki bakal update field kartu yang field pertamanya sama persis — jadi gak bikin duplikat 🎉

Cara 2 — Round-trip lewat tool ini (pakai GUID):

  1. Pas convert apkg→csv, centang dulu opsi "Sertakan GUID" di atas sebelum download CSV-nya
  2. Edit isi kartunya di CSV/preview, tapi JANGAN hapus/ubah kolom GUID-nya
  3. Convert balik CSV itu ke .apkg lewat mode "CSV → Apkg" — kartunya bakal otomatis pakai ID asli yang sama
  4. Import .apkg hasilnya ke Anki, tetap aktifin opsi update-by-match biar Anki nge-treat sebagai update, bukan kartu baru
Langkah 1 — Pilih file CSV-mu
🌸📊🌸

Baris pertama sebaiknya berisi nama kolom (Front, Back, dsb)
📄
Aturan format CSV-nya
  • Kolom pertama otomatis jadi sisi depan kartu, kolom sisanya jadi sisi belakang
  • Kolom bernama persis Tags otomatis dipakai sebagai tag Anki
  • Kolom bernama persis GUID (opsional, biasanya hasil dari mode "Apkg → CSV" tool ini dengan opsi "Sertakan GUID" dicentang) dipakai buat mempertahankan identitas kartu asli — cocok buat update-in-place
  • Kolom bernama persis Note Type (opsional) bikin tiap nilai unik di situ jadi note type-nya sendiri — jadi 1 file CSV bisa punya beberapa note type berbeda sekaligus
  • Note type baru dibuat otomatis sesuai jumlah & nama kolom CSV kamu (field yang kosong buat note type tertentu dibiarkan kosong)
  • Mendukung 2.000 – 10.000+ kartu, tinggal tunggu beberapa detik pas generate
